Полностью реализовать и интегрировать CONTENT HUB v2 + READER v1.0 ровно по спецификации в файле docs/AXIOM_CONTENT_HUB_v2_SPEC.md (Часть 1 и Часть 2), а также вести подробный лог своих действий.

ОБЯЗАТЕЛЬНЫЙ ПОРЯДОК ДЕЙСТВИЙ:

1. Сначала:
   - Открой и внимательно прочитай файл docs/AXIOM_CONTENT_HUB_v2_SPEC.md целиком.
   - Особое внимание удели:
     - разделам 2–6 (архитектура данных, build-скрипт, CONTENT HUB, READER, миграция),
     - разделу 9 «ПОДРОБНЫЕ ЧЕК-ЛИСТЫ ПО БЛОКАМ» — это твой основной рабочий чек-лист,
     - разделам 10–11 — формат лога и блок, откуда ты начинаешь вести записи.

2. Относись к docs/AXIOM_CONTENT_HUB_v2_SPEC.md как к ЗАКОНОДАТЕЛЬНОМУ ФАЙЛУ:
   - Нельзя менять формулировки пунктов ТЗ и структуру документа.
   - Разрешено:
     - отмечать чекбоксы в разделе 9 (переход `[ ]` → `[x]` и обновление статусов),
     - добавлять записи лога ТОЛЬКО ниже маркера `<!-- AXIOM_AGENT_LOG_START -->` в конце файла, если это явно предусмотрено,
     - при необходимости вносить минимальные техправки (опечатки, неверные ссылки), обязательно описывая это в логе.

3. Создай и используй лог-файл:
   - Основной лог: docs/AXIOM_CONTENT_HUB_v2_LOG.md (относительно ROOT).
   - Оформи его по шаблону из раздела 10 SPEC-файла:
     - шапка,
     - блок GLOBAL STATUS,
     - блоки STEP LOG для каждого шага,
     - финальный SUMMARY.
   - Все записи в логе делай в формате из раздела 10.4 (STEP LOG) с указанием:
     - Status,
     - Date,
     - Description,
     - Files created / changed / removed (обязательно относительные пути от ROOT),
     - Notes / Problems при необходимости.

4. Работай ПО БЛОКАМ A–F (раздел 8 и 9 SPEC):
   - Блок A — данные и исходники (content-src, public/content-html, content-index.json).
   - Блок B — скрипт сборки контента (scripts/build-content.*, package.json).
   - Блок C — CONTENT HUB UI (Sidebar, Preview, ContentHubPage).
   - Блок D — READER UI (ReaderPage, загрузка HTML, роутинг).
   - Блок E — миграция/очистка старого CONTENT.
   - Блок F — проверка и финальное подтверждение.
   - Разрешено идти последовательно A→F или брать логически связанные блоки группами, но в логе всегда фиксируй, какой блок и подпункты выполнялись.

5. Для КАЖДОГО СУЩЕСТВЕННОГО ШАГА:
   - Обнови чек-лист в разделе 9 SPEC-файла (если это уместно для пункта).
   - Добавь запись `STEP LOG` в docs/AXIOM_CONTENT_HUB_v2_LOG.md:
     - Укажи точный идентификатор шага (например: `STEP: A.1 · content-src — создание и наполнение`).
     - Явно перечисли:
       - Files created (с путями),
       - Files changed (с путями),
       - Files removed (с путями),
       - Краткое описание изменений.
   - Всегда сохраняй соответствие между пунктами ТЗ и своими шагами в логе (по кодам A.1, B.3, C.4 и т.д.).

6. При изменении кода:
   - Всегда указывай относительные пути файлов от ROOT.
   - Соблюдай архитектуру и маршрутизацию, описанные в ТЗ (каталоги `content-src/`, `public/content-html/`, `src/features/content/...`, `scripts/...`).
   - Внимательно следи за тем, чтобы:
     - CONTENT HUB рендерился через новый `ContentHubPage`,
     - READER работал по маршруту `/content/:id` (или фактическому эквиваленту в роутере),
     - build-процесс `npm run build` (или аналог) завершался успешно после интеграции скрипта `build:content`.

7. Если какой-то пункт ТЗ невыполним в текущих условиях:
   - Не придумывай альтернативную архитектуру самостоятельно.
   - Пометь соответствующий пункт как `BLOCKED` или `SKIPPED`:
     - в чек-листе (раздел 9),
     - в логе `AXIOM_CONTENT_HUB_v2_LOG.md` (описать причину, что мешает выполнению).
   - Если у тебя есть предложение по решению — добавь его в `Notes` для этого шага или в отдельный файл `docs/AXIOM_CONTENT_HUB_v2_ISSUES.md`, если он существует.

8. Коммиты:
   - Формируй понятные commit message, связанные с блоками:
     - `feat(content): implement ContentHubPage`,
     - `feat(reader): add ReaderPage and routing`,
     - `chore(content-build): add build-content script` и т.п.
   - В логе всегда указывай, какой блок/шаг был покрыт данным коммитом.

ТВОЯ ЦЕЛЬ:
Используя ТОЛЬКО SPEC-файл (docs/AXIOM_CONTENT_HUB_v2_SPEC.md) и структуру репозитория (корень /e/ENDEAVOR/AXIOM WEB CORE/ui/), автономно:
- построить целостный модуль CONTENT HUB v2 + READER,
